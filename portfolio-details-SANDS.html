SANDS—Semi-Automated Non-response Detection for Surveys

Surveys, administrative records, and web scraping generate massive amounts of free-text responses, many of which require manual review to distinguish between valid responses and item non-response (e.g., "don’t know," "skip," or blank text). I led a team that developed SANDS (Semi-Automated Nonresponse Detection for Survey Text), a machine learning model designed to automate this review process, significantly reducing the need for manual coding.

SANDS, the CDC’s first publicly released machine learning model, significantly reduces human effort in data cleaning and analysis, making it easier to process qualitative and text data from sources such as survey responses, hospital notes, and Electronic Health Records (EHRs). By leveraging natural language processing (NLP) and supervised learning, this model improves efficiency in handling large-scale qualitative datasets.

Goals

  • Develop a machine learning model that can automatically classify text responses as either valid data or item non-response.
  • Reduce manual effort and subjectivity in cleaning large-scale qualitative datasets.
  • Ensure the model is transparent, explainable, and publicly available for research use.
  • Evaluate the model across national surveys, health records, and administrative datasets to improve data quality and analysis efficiency.

Challenges

  • Handling highly imbalanced data, as explicit blanks or missing responses are rare. Instead, non-response is context-dependent, requiring consideration of both the data source and analytical goals to classify text accurately.
  • Ensuring high accuracy and generalizability across different types of surveys and records.
  • Addressing ethical and transparency concerns by developing a clear model card and public documentation.
  • Gaining approval for public release as CDC’s first openly available ML model.

Methods

  • Natural Language Processing (NLP): Fine-tuned a pre-trained BERT language model for response classification, optimizing it to distinguish between valid responses and item non-response.
  • Supervised Learning: Labeled datasets were used to train a binary classifier that distinguishes valid responses from nonresponses.
  • Feature Engineering: Incorporated both semantic meaning (via embeddings) and text structure patterns to improve classification accuracy.
  • Model Evaluation: Conducted rigorous accuracy testing, bias detection, and generalization analysis across multiple datasets.

Outcomes & Impact

  • Released and deployed SANDS, now publicly available on Hugging Face, marking CDC’s first machine learning model release.
  • Demonstrated a high accuracy rate in detecting item non-response, significantly reducing manual coding effort.
  • Improved the efficiency of survey processing, administrative records analysis, and health data research.
  • Enabled greater scalability and reproducibility for government and academic researchers working with large qualitative datasets.
